# Model

The distributed result of training.

The model binary is **not stored in this repository** — it ships as a download
asset on the [**Releases**](../../releases) page. The repo holds only the model's
documentation and evaluation output.

## Getting the model

Download `cu.traineddata` from the latest release, then put it where Tesseract
looks for language data and use `-l cu`:

```bash
# grab the asset (via the Releases page, or the gh CLI):
gh release download --repo <owner>/<repo> --pattern cu.traineddata

# install it, or keep it anywhere and point at it with --tessdata-dir
cp cu.traineddata "$(dirname "$(command -v tesseract)")/../share/tesseract-ocr/"*"/tessdata/"
tesseract line.png stdout --psm 13 -l cu       # raw-line mode for single-line crops
# …or without installing:
tesseract line.png stdout --psm 13 -l cu --tessdata-dir /path/to/download-dir
```

> Keep the asset named exactly `cu.traineddata` — Tesseract derives the `-l cu`
> language name from the filename. The release **tag** carries the version
> (e.g. `v1.0`); the filename stays constant so it's drop-in.

## Publishing a new model

From a local `make train` (which leaves `model/cu.traineddata` in place):

```bash
make release VERSION=v1.0        # gh release create v1.0 model/cu.traineddata …
```

Update `MODEL_CARD.md` in the same release so metrics and provenance match the
binary.

## Also here

- `MODEL_CARD.md` — provenance, training data, metrics, intended use, and limits.
  **Read this before relying on the model.**
- `eval/` — evaluation output (`metrics.tsv`, `report.html`, `review.png`),
  regenerated by `make eval`.

The training scratch (checkpoints, unicharset, proto model) lives in `../training/`
and is not committed.
